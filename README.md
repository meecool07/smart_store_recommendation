## Why FP-Growth over Apriori?
| Feature | Apriori | FP-Growth |
|---------|---------|-----------|
| Database scans | Once per itemset size | Only twice |
| Memory usage | Exponential candidates | Compact FP-tree |
| Speed | Slow | 10-100x faster |
